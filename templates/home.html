About

This is a project is a solution for the Full-stack Software Engineer test by PackForm

The objective of this project is to develop a web application which allows the company representative to store and view orders data

On the first version we have two api end points

  • Orders: Can be used for viewing order book
  • Upload: Can be used for uploading order data

Tech

The core tech used in this project is Flask Framework, MongoDB and PostgreSQL

Flask Framework is a python based restful api framework which is used to create api end points and python is used for backend

Atlas MongoDB is a cloud based nosql database which is used to store Customers data

Heroky PostgreSQL is a cloud based sql RDBMS database which is used to store Orders and Delivery

Data Upload

The orders data which would be uploaded using the uploaded endpoint allows user to upload any of the 5 possible input files in csv format

  1. Orders: Csv file with columns id,created_at,order_name, and customer_id. On sucessfull upload this data would be stored in PostgreSQL server

  2. OrderItems: id,order_id,price_per_unit,quantity,product. On sucessfull upload this data would be stored in PostgreSQL server

  3. Deliveries: Csv file with columns id,order_item_id,delivered_quantity. On sucessfull upload this data would be stored in PostgreSQL server

  4. Customers: Csv file with columns user_id,login,password,name,company_id,credit_cards. On sucessfull upload this data would be stored in MongoDB server

  5. Customer Companies: Csv file with company_id,company_name. On sucessfull upload this data would be stored in MongoDB server
